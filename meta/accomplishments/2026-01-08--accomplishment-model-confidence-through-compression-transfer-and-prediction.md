## Coherence checks already verified across domains (high-signal list)

### 1) Predicts failure modes in advance (not only post-hoc)

**A. “Trying to change experience from inside the distressed self-model reinforces it.”**

- Urge to solve/change/check arises inside a negative self-embedding.
- Acting from that urge worsens downstream (more salience, compulsion, DP/DR).
- Switching to a chosen belief held by another self-model (or compassionate observer) changes downstream interpretation and behavior.

Key marker: effort helps only when authored by a different interpretive frame; “trying harder” makes it worse when authored by the same self-model generating distress.

**B. “Evaluation triggers the downshift.”**

- Momentum flows → inventory-taking / evaluation → instant loss / self-model flip.
- Downshift is not diffuse fatigue; it is attribution collapse (“that wasn’t really me / doesn’t count”).

Key marker: a consistent, specific cognitive operation functions as the causal switch (evaluation → identity attribution collapse), not vague stress.


---

### 2) Unifies DP/DR, OCD-like loops, shame spirals, and motivation under one control architecture

**A. OCD loop fits DP/DR**

- OCD: threat tag → urge to neutralize → reinforcement; themes vary but role in loop is invariant.
- DP/DR: threat to identity/realness/continuity → checking self/reality (“is this real / am I okay?”) → reinforcement.

Mechanistic claim: same loop, different object.

**B. Motivation uses the same architecture with a positive target**

- “Am I enjoying yet?” checking behaves like compulsion.
- Failed check produces shame/loss.
- Intervention: stop checking + hold belief “enjoyment will come” + act values-based.

Mechanistic claim: motivation is not willpower; it is simulation + belief + non-checking (the same control loop without threat content).

### 3) Two-stage loving-kindness maps onto identity/affect split and predicts time course

**Stage 1: identity constancy via compassionate observer**

- Observer stance stabilizes self-image and reduces moral contamination/judgment.
- Shifts attention from fluctuating state to stable self-anchor.

**Stage 2: affect (“little flame”) as a permission gate**

- Affect requires belief in a feeling not currently felt.
- “Hold it as already there; don’t check; allow flow.”

**Time-course invariant**

- Warmth tends to arise days after identity anchoring—consistent with stabilization first, affective reconsolidation later.

### 4) “Chosen belief + held context” explains reconsolidation and lived integration events (including latency)

**Recipe observed repeatedly**

- Bring memory online (destabilize).
- Hold disconfirming stance/belief.
- Actively bring confirming context/details into working memory.
- Repeat through “extinction burst” until reweighting occurs.
- Later: integration event / deep sleep / baseline shift.

Key marker: latency effects and post-event consolidation patterns; weak models predict immediate relief, this one predicts delayed emergence.

**Commit-message overlap**

- End-of-sprint framing acts like a “commit message” that tags memory for consolidation.
- Achievement framing stabilizes; shame framing destabilizes.

Key marker: cross-domain overlap between software mental model (commit message), reconsolidation re-tagging, and direct behavioral verification.

### 5) Fierce self-compassion (FSC) slots in as the protective “probability-collapse” tool against shame-authored frames

Two “collapses” distinguished:

- Unhealthy collapse: shame/anger exporting certainty (“I’m bad / you’re bad”).
- Healthy collapse: FSC rejects false self-judgment (“no — I know who I am”) while preserving reality and learning.

Key marker: “no” is not force; it is permission to withdraw epistemic authority from the wrong frame while holding a truer one long enough for de-salience.

Internally: shame salience, DMN blame/urgency generation, compulsive solving reinforcing the active model.

Key marker: mechanism generalizes; it is not “about me,” it is a stability architecture appearing inside and outside.
